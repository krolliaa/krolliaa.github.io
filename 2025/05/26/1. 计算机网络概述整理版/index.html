计算机网络概述

什么是网络？

  • 定义：网络是一种结构，由节点（Node）和边（Edge）组成。
  • 本质：多个节点通过边相连使之形成一个互相之间可以通信的系统。
  • 举例：
    • 蜘蛛网（节点是网点，边是丝）
    • 社交网络（人是节点，关系是边）
    • 神经网络（神经元是节点，突触是边）

什么是计算机网络？

  • 定义：计算机网络是由连接起来的计算设备和中间交换设备组成的通信系统。
  • 目的是让主机之间能高效、安全、可靠地通信和资源共享。

构成要素：

1. 节点（Node）：

  • 主机节点（Host）（方形）：（端系统End Systems）
    • 最终用户的设备：如笔记本、手机、Web服务器、智能家居设备等。
    • 功能：数据的发送者/接收者，运行应用进程。
    • 特点：
      1. 数以亿计，组成“互联网的边缘”
      2. 是网络通信中的源头和重点
      3. 在主机上运行各种网络应用程序，与其他主机进行逻辑通信
  • 数据交换节点（圆形）：
    • 如路由器、交换机、负载均衡器等。
    • 功能：数据中转、转发，既不是源也不是目标。负责在网络中“转发数据分组（Packet）”的设备，不产生也不消费数据，只负责中转、转发。

2. 边（Edge）：

  • 通信链路（Links），是连接节点之间的数据通道。
    • 接入链路（Access Link）：
      • 连接主机与接入路由器（主机节点 → 数据交换节点）
    • 主干链路（Core Link）：
      • 连接路由器之间（数据交换节点 ↔ 数据交换节点）

3. 协议：

  • 协议是网络运行的“语言”，即通信规则。
  • 定义：对等层的实体在通信中应遵守的规则，包括语法、语义（语法 + 语义 = 格式）和时序以及动作。
  • 不同厂商的设备通过遵守协议标准实现互操作性。
  • 分层结构：
    • 物理层、链路层、网络层、传输层、应用层
    • 典型协议：TCP、UDP（传输层），IP（网络层）

什么是互联网（Internet）？

  • 定义：互联网是一个全球性的计算机网络，它运行在一套以 TCP/IP协议族 为核心的标准之上。（运行或者说工作在以 TCP/IP 为主的那一簇协议为基础之上的那个网络就叫做互联网）。

  • 本质结构：从宏观来看，互联网是由数以百万计的子网络通过网络互联设备连接起来形成的“网络的网络”。

    • 小网络：家庭网、校园网、企业网

    • 大网络：中国电信、中国移动、教育网

    • 通过路由器连接形成更大的网络结构

    • 全球互通，任意连接 —— 无中心化的结构

与内部网（internet）的区别：

特征 Internet（互联网） internet（内部网）
协议 TCP/IP TCP/IP
是否连到全球互联网 是 否
用途 面向公众，全球互通 内部使用（如企业、校园）
网络类型 公共网络 私有网络

互联网的发展标准：IETF 与 RFC

IETF（Internet Engineering Task Force）：

  • 非盈利组织，负责制定和推动互联网相关标准的发展

RFC（Request for Comments）：

  • 互联网标准的发布形式，任何人可提交草案

  • 经过评审、编号、公开发布，广泛采纳形成标准

  • 一个协议可能由多个 RFC 文档组成

  • 举例：TCP、IP、HTTP 都有相关的 RFC 文档

协议的开放性和互操作性

  • 不同厂商的设备或操作系统（如 Android、Linux、iOS）之间能通信，是因为它们都遵守相同协议标准

  • 协议是全球共识的语言，就像人类交流需要通用语言（如英语）

从另外一个角度看互联网

互联网 = 分布式应用进程 + 提供通信服务的基础设施

  1. 分布式应用进程

    • 互联网的本质之一是由运行在不同主机上的多个应用进程组成的分布式系统。
    • 举例：网页浏览、电子邮件、在线视频、在线游戏等，都是分布式应用。
    • 这些应用进程通过互联网进行交互、协作，完成具体的任务和服务。

  2. 提供通信服务的基础设施

    • 这部分是支撑分布式应用能够正常通信的“底层结构”。

    • 包括：

      • 应用层以下的所有网络协议（如 TCP、UDP、IP 等）和协议实体；

      • 各种网络设备和结构（如路由器、交换机、链路）；

      • 各主机中，应用层以下的操作系统网络模块、协议栈等；

      • 为应用提供的编程接口（API），即应用访问网络服务的方式。主要有两类：

        • 面向连接的服务（如 TCP 提供的服务）：在通信前建立连接，通信过程中保障顺序与可靠性。

        • 无连接的服务（如 UDP 提供的服务）：不建立连接，数据包独立发送，不保证顺序与可靠性。
