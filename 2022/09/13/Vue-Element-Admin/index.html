vue-element-admin

1. 简介

vue-element-admin是基于element-ui的一套后台管理系统集成方案。

  • 官网网址：https://panjiachen.github.io/vue-element-admin-site/zh/guide/

2. 安装

注：安装时可能因为nodejs npm版本问题导致无法安装，此时建议升级版本或者卸载安装高版本的nodejs即可

注：这里使用新版本的admin npm i一直卡住，建议使用别的vue-element-admin版本

3. vue-Admin-template

vue-Admin-template是基于vue-element-admin的一套后台管理系统基础模板（最少精简版），可作为模板进行二次开发。

建议：你可以在 vue-admin-template 的基础上进行二次开发，把 vue-element-admin当做工具箱，想要什么功能或者组件就去 vue-element-admin 那里复制过来。

安装操作跟第二步一致。

新版vue-element-template正常。

整个逻辑推导过程如下：

  1. 输入npm run dev启动项目，然后dev配置是在package.json文件中配置的，于是可以找到启动的配置文件在build/webpack.dev.conf.js
  2. 于是找到build/webpack.dev.conf.js文件，最上方可以看到引用了基础的webpack配置文件：const baseWebpackConfig = require('./webpack.base.conf')所以就又可以找到build//webpack.base.conf文件
  3. 于是找到build/webpack.base.conf，当初在学习打包的时候使用的webpack.config.js文件中在entry{app}中配置了入口文件，于是就可以找到入口文件src/main.js
  4. 于是找到src/main.js项目入口文件，可以看到定义了Vue实例
  5. 通过在web中打开index.html页面，可以知道引入了app.js文件，说明webpack将main.js打包后输出成了app.js，那webpack是如何自动整合输出为index.html的呢？原因就在于在build/webpack.dev.conf.js打包工具的配置文件中，引入了HtmlWebpackPlugin插件，所以才可以将html跟app.js整合在一块
  6. 再从HtmlWebpackPlugin插件的上方可以看到引入了process.env，于是找到config/dev.env文件，可以看到这是学习axios分层的时候的第一部分，定义BASE_API基础URL地址。
  7. 那分层的第二部分axios.create在哪里呢？仔细找找在src/utils/request.js中
  8. 真正的发起请求就在src/api/login.js或者src/api/table.js中

以上就是整个前端项目从项目运行到向后端发起请求的整个过程。

1. package.js

npm项目的核心配置文件，包含项目信息，项目依赖，项目启动相关脚本

  • 启动项目的命令： npm run dev

  • dev脚本：webpack-dev-server --inline --progress --config build/webpack.dev.conf.js

    • webpack-dev-server：一个小型的基于Node.js的http服务器，可以运行前端项目
    • --inline：一种启动模式
    • --progress：显示启动进度
    • --config build/webpack.dev.conf.js：指定webpack配置文件所在位置

2. build/webpack.dev.conf.js

webpack配置文件，包含项目在开发环境打包和运行的相关配置

  • webpack.dev.conf.js中引用了webpack.base.conf.js
  • webpack.base.conf.js中定义了项目打包的入口文件

因此生成的js文件会被自动插入到名为index.html的页面中

3. index.html

项目默认的html页面

4. src/main.js

项目js入口文件，项目的所有前端功能都在这个文件中引入和定义，并初始化全局的Vue对象

5. config/dev.env.js

定义全局常量值

因此，在项目中的任意位置可以直接使用process.env.BASE_API常量表示后端接口的主机地址

6. src/utils/request.js

引入axios模块，定义全局的axios实例，并导出模块

7. src/api/login.js

引用request模块，调用远程api
