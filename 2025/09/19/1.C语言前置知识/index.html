1.C语言前置知识

前置知识

编程思维

把大象塞进冰箱要几步？相信很多人听说过这个问题，答案是三步：

  1. 打开冰箱
  2. 把大象塞进冰箱
  3. 关上冰箱

计算机也是这么做的，这就是编程 —— 用计算机能听懂的语言，一步步告诉它该怎么做。

你只需要一个写代码的工具（编辑器）以及把一个处理代码的“核心工具”。这个核心工具根据语言类型，可能是“编译器”或“解释器”（后面会提到）。你就可以开始编程大业了。😀

计算机的问题求解方法

比如说要解方程：2x+6=20，x在1到10之间。人一眼就能看出x=7，但计算机得用笨办法：

方法一：傻傻地试（枚举法）

  1. x=1？不对
  2. x=2？不对
  3. x=3？不对
  4. …
  5. 一直试到x=7，搞定

方法二：聪明一点的尝试（二分法）

  1. 先试中间值x=5，小了

  2. 再试x=7.5，大了

  3. 那就试x=6.5，还是小了

  4. 最后锁定x=7，成功

程序执行机制

两种方式：

  • 现场翻译（解释执行）：有个翻译官在现场，你说一句，他马上翻译给计算机听，计算机就做一句。（通过解释器实时解析和执行源代码）
  • 全文翻译（编译执行）：先把你的话全部翻译成计算机母语，写成说明书，计算机直接照着做。（将源代码编译为机器码后直接执行）

两种方式各有优势：解释执行提供更好的灵活性，编译执行则能获得更优的性能表现。

C语言应用价值

不是因为最简单，而是因为：

  1. 很多重要活儿只能 C 语言干 —— 比如操作系统、驱动程序这些底层工作。
  2. C 语言是很多语言的大哥 —— 学会了C，再学C++、Java、C#都会很轻松。
  3. 真正重要的是思路 —— 语言只是工具，解决问题的思路才是核心。

C 语言主要应用于以下领域：操作系统、嵌入式系统、设备驱动程序、图形引擎、图像处理和音频处理等底层开发。

作为系统级编程语言，C 语言更注重学习过程和开发效率，而非开发乐趣。

C语言发展历程

C 语言今年已经年过半百了，从 1969 年诞生到现在，它经历了：

  • 1973 年：第一次用来写 Unix 操作系统
  • 1989 年：有了第一个官方标准（ANSI C）
  • 1999 年：更新到现在最常用的 C99 标准

老当益壮，现在还在很多重要领域发挥着作用。

学习C语言心法

记住这句话：学习 C 语言主要是在做练习题，而不是开发软件。

就像学开车先要在驾校练习，而不是直接上高速公路。你会写很多小程序来理解，例如：

  • 计算机怎么计算数学题？
  • 怎么做判断？
  • 怎么重复做事情？
  • 怎么管理内存？
