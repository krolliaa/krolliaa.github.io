Vue3.0

第一个程序

  1. 引入 vue.js 核心文件
  2. 创建一个 div 标签 —> 挂载点
  3. 使用 Vue 对象调用 createApp 方法来创建 Vue 实例，再调用 mount 方法讲应用挂在到页面上【div 标签】

跟之前的 Vue 2.0 的写法有点不太一样。这里创建一个 Vue 实例直接封装成了 createApp 方法。

{{}} 表示插值表达式，可以直接写，而且插值表达式里面可以做运算。可以使用函数。【toFixed 的意思是保留小数点多少位】

Vue3.0 指令

v-text 指令

作用：跟插值表达式一样都是给数据设置文本值，插值表达式适用于填充部分内容

v-html 指令

v-html 指令跟 v-text 指令不同之处在于，v-html 指令在有 HTML 标签的时候会解析成标签，但是 v-text 只会将其解析成文本：

v-bind 指令

v-bind 的作用就是用于绑定元素的属性，比如 src title class href 等等，都是元素内部的属性，就可以使用 v-bind 绑定 data 中的值。v-bind 可以简写为：冒号。

v-bind 指令除了直接设置属性之外，还可以用来调控属性，使用{}就可以用来判断到底用不用，如果类名和调控可见的变量名一致，可以写单个名字。

v-on 指令

v-on 指令的作用就是可以为元素绑定事件，v-bind 可以简写为 : v-on 自然也可以简写，可以简写为 @，绑定的方法定义在 methods 属性中，data 是用来定义数据的，methods 是用来定义方法的，在 methods 定义的方法可以直接使用 this 来访问定义在 data 中的数据。methods 是一个对象，跟 data 同级。

图片切换案例

v-if 指令

v-if 用于决定是否显示元素，类似的还有 v-show ，只不过 v-if 是通过删除 DOM 节点进行显示 OR 不显示，v-show 只是通过 display: none; 隐藏起来而已。

v-else 指令

通常跟 v-if 联合使用

v-for 指令

v-for 的指令就是循环遍历，key 属性可以绑定 index，提高 DOM 节点的操作性能

v-model 指令

用于数据的双向绑定，只能用于表单元素，用在其它地方没有意义。

CompsitionAPI

CompsitionAPI 是 Vue3.0 独有的，Vue2.0 是 OptionsAPI，Vue3.0 也有 OptionsAPI，OptionsAPI 就是跟我们之前定义的方式是一样的，就是 data methods 分开来定义，如果代码比较短那还好，但是代码很长的话，像是 data 里面定义的数据你要找再哪个地方使用到就变得非常麻烦，看得眼花缭乱，但是 CompositionAPI 将各个功能单独放在一块。做好了这个功能之后再去做下一块。

Vue3.0 生命周期

一共有三个阶段：挂载、更新、销毁三个阶段。生命周期是在某一时刻自动运行，学用得上的就行。

挂载阶段：

  • beforeCreate：创建之前
  • created：创建完成，一般用于初始化页面需要的数据【用得上，AJAX 获取请求】
  • beforeMount：挂载之前
  • Mounted：挂载完成，挂载完成之后可以直接操作DOM对象【用得上】

更新阶段：

  • beforeUpdate：更新之前
  • update：更新完成

销毁阶段

CompositionAPI 取消了 beforeCreate 和 created，直接使用 setUp() 即可。
