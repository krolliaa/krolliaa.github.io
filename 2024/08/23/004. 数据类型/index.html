数据类型

什么是数据类型？

数据类型就是编程语言用来定义变量能够存储数据的数据种类的一个概念。

数据类型的作用

数据类型有三个方面的作用：

  1. 一是用来告诉Java该数据在内存当中要分配多大的内存空间
  2. 二是用来决定该数据的有效操作有哪些，比如说某个数据的数据类型是布尔类型的，那么该数据就无法进行算术运算而只能进行逻辑运算
  3. 三是用来提高程序的稳定性和鲁棒性，因为Java是一门强类型的编程语言，在编译的时候，编译器会检查各个不同数据类型的数据的兼容性，从而能够提前避免需要错误，例如int复制给string类型的变量，此时就会报错，这样就能提前避免。

数据类型的分类

在Java语言当中，数据类型可以被分为基本数据类型以及引用数据类型两大类。我们分别来说说：

  • 基本数据类型：直接存储数据的值

    • 基本数据类型包括：

      • 整数型（整数型的字面量的默认数据类型为int）

        • byte：默认值为0，占用1个字节，取值范围为-128-127
        • short ：默认值为0，占用2个字节，取值范围为-32768-32767
        • int：默认值为0，占用4个字节，取值范围为-2^31-2^31-1
        • long：默认值为0l/L，占用8个字节，取值范围为-2^63-2^63-1

        注：所以当使用long数据类型定义的字面两需要在后面加上l或者L

      • 浮点型（浮点型的字面量默认数据类型为double）

        • float：默认值为0.0f/F，占用4个字节，大约有效数字为6到7位
        • ``double`：默认值为0.0，占用8个字节，大约有效数字为15到16位

        注：所以当使用float数据类型定义的字面两需要在后面加上f或者F

      • 字符型

        • char：默认值为\u0000即空字符，Unicode字符集的第一个字符，占用2个字节，取值范围没有负数，为0-65535

      • 布尔型

        • boolean：只有true跟false两个值，注意，是只有

  • 引用数据类型：不直接存储数据的值而是存储指向对象的引用，包括数组、接口、类、String等等，默认值为null，表示没有任何引用。

数据类型的转换

数据类型的转换指的是不同数据类型之间进行相互转换的过程。

在Java中数据类型转换包括两种，一种是隐式类型转换也叫自动类型转换，一种叫做显示类型转换也叫强制类型转换。

  • 隐式类型转换（自动类型转换）：
    • 即小容量数据类型转换为大容量数据类型
      • 因为小容量的数据类型转换为大容量的数据类型可以安全地被大容量的数据类型所容纳因而不会造成数据上或者精度上的丢失。
  • 显式类型转换（强制类型转换）：
    • 即大容量数据类型转换为小容量数据类型
      • 因为大容量数据类型转换为小容量数据类型，小容量的数据类型容纳不下，可能会造成精度上或者数据上的丢失，因而需要强制（手动）的进行类型转换。

注意：复合赋值运算即：+= -= *= /=包含了隐式的强制类型转换，转换类型最终以起始变量为主。

数据类型的提升

  • byte short char：当这些数据类型参与运算的时候，Java会自动将它们提升为int数据类型，这是因为int数据类型是Java中最常用的数据类型，可以处理绝大多数的整数运算，避免了潜在的溢出问题。

    【注：除了这一项，其余的都可以总结为：提升为表达式中容量最大的数据类型】

  • int：如果所有操作都是int数据类型，那么运算结果也保持为int数据类型，不会自动提升。

  • long：如果表达式中有 long 类型的操作数，所有较小类型（如 int、short）都会被提升为 long，运算结果也是 long 类型。

  • float：如果表达式中有 float 类型的操作数，所有较小类型（如 int、long）都会被提升为 float，运算结果也是 float 类型。

  • double：如果表达式中有 double 类型的操作数，所有其他类型（如 float、int、long）都会被提升为 double，运算结果也是 double 类型。

  • 类型提升不仅存在于运算当中，比较当中也存在类型提升。即表达式当中。

数据类型与包装类

在Java中，每一个基本数据类型都对应着一个包装类Wrapper Class。这些包装类主要用于将基本数据类型转换为对象，这样可以利用Java的对象特性，比如存储在集合中，传递对象引用等。

整数类型

  • byte → Byte
  • short → Short
  • int → Integer
  • long → Long

浮点类型

  • float → Float
  • double → Double

字符类型

  • char → Character

布尔类型

  • boolean → Boolean

这里最主要的是说一下自动装箱（xxx.valueOf()）和自动拆箱xxx.xxxValue()以及==跟equals方法之间的相关问题。

  • 自动拆箱的时机：（即什么时候会发生自动拆箱）

    • 参与算术运算时，会先自动拆箱成基本数据类型然后进行运算 —> 更根本底层逻辑：只有基本数据类型可以直接参与算术运算跟逻辑运算，因此需要自动拆箱
    • 使用==运算符比较时，使用==时表明需要比较的是值因此会进行自动拆箱
    • 在条件语句中，也会进行自动拆箱以进行逻辑运算（本质上也是基本数据类型才能直接参与算术预算或者逻辑运算）
    • 方法调用时，如果方法中的参数时基本数据类型，而你传递的时包装类，那么Java会帮你自动拆箱
    • 三元运算

    注：注意null防止包装类为空这种特殊情况，只有再不为null的情况下的算术运算跟逻辑运算才是有效的，否则是无意义的，总而言之，记住：除了方法传递那一个，记住只有基本数据类型才能直接参与算术或者逻辑运算包括表达式的==比较以及条件语句、三元运算等等

  • 自动装箱的时机：（即什么时候会发生自动装箱）

    • 赋值时：当你将基本数据类型赋值给包装类变量时
    • 方法调用时：方法参数是包装类但是传递的是基本数据类型的参数
    • 在集合类中：集合类只能存储对象，无法存储基本数据类型因此会自动装箱
    • 方法返回值

注：频繁的拆箱跟装箱会影响程序性能

  • 特殊缓存机制：在编写代码的过程中，小整型数据是非常常用、高频的，所以Java为了减少内存的分配以及减少垃圾回收的开销，更进一步地提升效率，所以就会将小整型数据，特指[-128 - 127]之间地数据缓存起来。这种缓存机制见于多个包装类之中：
    • Byte：缓存了[-128 - 127]
    • Short：缓存了[-128 - 127]
    • Character：【特殊】因为取值范围只有且只能是正数，所以缓存的是[0 - 127]
    • Integer：缓存了[-128 - 127]
    • Long：缓存了[-128 - 127]
  • Boolean包装类更加特殊一些，只有两个值且也是被缓存的：true和false。
  • 目的：高频使用所以减少内存分配以及垃圾回收的开销，提高效率。

6条规则

  1. 布尔类型的数据无法进行算术运算，只能进行逻辑运算，且只有布尔类型的数据是无法进行数据类型的转换的，其它类型的数据也无法转换成布尔类型。

  2. byte[-128 - 127] short[-32768 - 32767] char[0 - 65535]都是经常会使用到的数据类型，所以当整数型字面量在它们的取值范围之内时，都是被Java所允许可以直接赋值的，编译器会帮助我们进行优化，这样做可以提高效率并且更加简洁。

  3. 数据类型容量大小排序且可以进行数据类型转换的排序为：byte < short(char) < int < long < float < double，char数据类型虽然跟short一样都占用2个字节但是char因为只有正数所以可以表示更大的正整数，取值范围为[0 - 65535]。

  4. 小容量转换为大容量的数据类型因为可以被大容量数据而类型所包容所以不会导致数据/精度上的丢失，所以可以自动转换，但是反之因为小容量无法包含所以可能会造成精度上或者数据上的损失，因而需要强制类型转换，使用的时候也应当保持谨慎。

  5. byte short char在进行计算的时候，表达式分两种情况一种是字面量表达式，另外一种则是变量表达式，字面量表达式在编译阶段就可以计算得到结果了，而变量表达式因为变量是动态的不确定的，所以无法在编译阶段就确定结果如何，需要在运行时阶段才能确定，而又因为变量是动态的，其变量的值在编译时不确定，所以Java编译器为了保守起见会将变量都默认提升为int数据类型（若表达式中存在更大容量的数据类型则会提升为该最大的数据类型即：以表达式中存在的最大的数据类型为基准）。这样做的目的不仅提高了效率而且确保了整体代码的安全性。所以说如果是直接字面量的表达式例如：

习题：

  1. byte b1 = 1000; //错误，超过了 byte 数据类型的取值范围，需要强制类型转换
//修改 --- 此时就会按照 1000%256 = 232 --- 232>127 --- 232-256 = -24 --- 结果为-24
byte b1 = (byte) 1000;


  2. short s = 1000; //正确，没有超出 short 的取值范围，编译器会优化进行自动类型转换


  3. int f = 10 / 3; //正确，结果为 3
long g = 10; //正确，虽然整数型字面量的数据类型默认为 int 数据类型，但是这里小容量赋值给了大容量数据类型，会进行自动类型转换，后面的 l 跟 L 其实是可选的，除非你要明确表示这个数字是 long 类型的或者已经超过了 int 数据类型的范围
int h = g / 3; //不正确，因为 g 是 long 数据类型的，g / 3 得到的结果是 long 数据类型的，大容量赋值给小容量可能会出现数据或者精度上的丢失，所以需要进行强制类型转换
// 修改
int h = (int)(g / 3);
long m = g / 3; //正确，因为 g / 3 得到的结果就是 long 数据类型的，m 也是 long 类型，是对的
byte x = (byte)g / 3; //错误，虽然 g 强制转换为了 byte 数据类型，但是在进行运算的时候，类型会自动提升为 int 数据类型，所以得到的结果是 int 数据类型。
// 修改
byte x = (byte)(g / 3);
short y = (short)(g / 3); //同上，正确


  4. char cc = 'a'; //正确
System.out.println("cc = " + cc); //正确
System.out.println((byte)cc);//正确，97
int o = cc + 100;//正确，197
System.out.println(o);//正确，197


  5. char 类型变量能不能存储一个汉字？为什么？
回答：可以，因为 char 类型占用 2 个字节，而一个汉字刚好也占用 2 个字节，所以是可以存储一个汉字的。

// 错误，跟第十题的理由差不多，这里再详细说明下，2147483648 是整数型字面量，但是它已经超过了 int [-2^31 - 2^31]这个范围，int 已经装不下了，所以需要转换成 long 数据类型，修改如下：
//注意：修改的错误写法 ---> 因为编译器会优先识别字面量，这里字面量已经是超出 int 数据类型了，所以即使前面有强制类型转换也依然会出现编译错误
long a = (long)2147483648;
//第一种：
long a = 2147483648l;
//第二种：
long a = 2147483648L;


  7. char c = 65536;
//错误，char 数据类型的容量在[0 - 65535]之间，超出了这个范围属于 int 数据类型，需要强制转型。修改如下：
char c = (char) 65536;
